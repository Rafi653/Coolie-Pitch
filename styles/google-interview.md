# Google Interview Process

## Overview
Google's interview process is known for being rigorous and comprehensive, with strong emphasis on algorithms, data structures, and problem-solving skills.

## Interview Rounds

### 1. Phone Screen (45 minutes)
- 1-2 coding questions
- Medium difficulty typically
- Conducted via Google Meet with shared doc
- Focus on clean, working code

### 2. Onsite/Virtual Onsite (4-5 rounds, 45 min each)

#### Coding Rounds (2-3 interviews)
- Algorithm and data structure problems
- Medium to Hard difficulty
- Must write production-quality code
- Optimize for time and space complexity
- Consider edge cases

#### System Design (1 round)
- Design scalable systems
- For senior roles: more complex systems
- Discuss tradeoffs
- Consider real-world constraints

#### Googleyness & Leadership (1 round)
- Behavioral questions
- Cultural fit assessment
- Past experiences and collaboration
- Leadership examples

## What They Look For

### Technical Skills
- Strong CS fundamentals
- Clean, efficient code
- Problem-solving approach
- Optimization abilities

### Communication
- Think out loud
- Ask clarifying questions
- Explain your approach
- Discuss tradeoffs

### Googleyness
- Comfort with ambiguity
- Bias to action
- Collaborative mindset
- User focus

## Common Topics

### Data Structures
- Arrays, Strings
- Hash Tables
- Trees (Binary, BST, Tries)
- Graphs
- Heaps
- Linked Lists

### Algorithms
- Sorting and Searching
- Dynamic Programming
- Recursion
- Breadth-First Search
- Depth-First Search
- Two Pointers
- Sliding Window

## Preparation Tips

1. **Practice on LeetCode**
   - Focus on Medium problems
   - Cover all major topics
   - Time yourself (45 min max per problem)

2. **Explain Your Thinking**
   - Don't code in silence
   - Discuss approach first
   - Mention optimizations

3. **Write Clean Code**
   - Use good variable names
   - Handle edge cases
   - Test your solution

4. **Study System Design**
   - Read "Designing Data-Intensive Applications"
   - Practice common designs (URL shortener, newsfeed, etc.)
   - Understand scaling concepts

5. **Prepare Stories**
   - Have 5-7 STAR format examples
   - Cover different situations
   - Emphasize impact

## Timeline
- Application to initial response: 1-2 weeks
- Phone screen scheduling: 1-2 weeks
- Onsite scheduling: 2-4 weeks after phone screen
- Final decision: 1-2 weeks after onsite
- **Total**: ~2-3 months typical
